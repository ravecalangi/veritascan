News Verification Engine

Image Detection Engine

AI Image Detection Engine AIDE

AIDE determines whether a submitted image was generated by an AI model (Midjourney, DALL·E, Stable Diffusion) or captured by a real camera. It combines the AI or Not API with pixel-level forensic signals to deliver a reliable, confidence-backed verdict.

I

AI or Not API — Core Detection Layer

Deep learning classifier · ~96% benchmarked accuracy

The primary detection signal comes from the AI or Not API — a purpose-built classifier trained on millions of AI-generated and authentic photographs. The image is base64-encoded and sent to the API endpoint, which returns a binary classification (AI-generated or Human-made) alongside a probability score.

91.7%

8.3%

API Verdict: AI_GENERATED — Confidence: 0.917

Scores above 0.85 are classified as AI-generated. Benchmarked at ~96% accuracy on Midjourney v5, DALL·E 3, and Stable Diffusion XL test sets.

How it works — The image is base64-encoded and submitted via POST to the AI or Not endpoint. The API's deep learning model returns a probability distribution over the two classes.
II

Pixel-Level Forensic Analysis

PRNU · ELA · Laplacian variance

AIDE performs three local forensic checks that exploit statistical fingerprints left by AI generative models versus real camera sensors.

Noise Pattern Analysis (Camera Sensor Fingerprint)

Authentic camera images contain Photo-Response Non-Uniformity (PRNU) — a unique noise pattern from manufacturing imperfections. AI images lack this signal entirely, producing near-zero noise floors detectable via FFT frequency-domain analysis.

Detected sensor noise variance 0.003 (near-zero)

Expected range in real photos: 0.04 – 0.18 variance

JPEG Compression Artifact Consistency

Real photographs accumulate compression artifacts in predictable DCT block patterns (8×8 grids). AI images saved as JPEG show inconsistent or pristine block structures — a statistical anomaly flagged by Error Level Analysis (ELA).

ELA inconsistency score HIGH (anomalous)
Texture Regularity & High-Frequency Content

Generative models (GANs, diffusion models) produce unnaturally smooth textures in backgrounds, hair, and fabric. Laplacian variance is measured per region; AI images score significantly lower than real photographs.

Submitted image (Laplacian variance)18.4
Typical real photo baseline96.0
Example — A real street photo → high sensor noise, natural DCT blocking, varied texture variance. A Midjourney portrait → near-zero sensor noise, smooth ELA map, unnaturally low Laplacian score.
III

Metadata & Provenance Checks

EXIF inspection · software tag detection

EXIF metadata is inspected for authenticity signals. Real camera images typically contain shutter speed, ISO, aperture, GPS coordinates, and a recognized camera model string. AI-generated images either lack EXIF data entirely or carry software-generated tags (e.g. "Adobe Firefly", "StableDiffusion").

Camera Model Not found
GPS / Location Data Absent
Software Tag "Midjourney v6"
Shutter / ISO / Aperture Not found

3 of 4 provenance checks failed. Missing camera metadata combined with a known AI software tag is a strong indicator of synthetic origin.

Example — Canon EOS R5 JPEG → full EXIF with GPS, lens model, shutter data. DALL·E export → no camera metadata, "Adobe Generative" software string.

Confidence Cap Enforcement

Hard-coded 3% – 97% clamp · Cannot be bypassed by AI

Even when an AI model returns 100% confidence, Veritascan's server enforces a hard clamp of 3–97% on all confidence values and credibility scores before they are displayed. This is applied at the code level — the AI model cannot bypass it regardless of prompt instructions.

3% Min
50% Uncertain
97% Max

Any value outside 3–97% is automatically clamped before display

90 – 97%Official announcement or high-confidence training knowledge
70 – 89%Strong credible evidence but no official announcement
50 – 69%Mixed or limited evidence
30 – 49%Mostly rumors or speculation
3 – 29%Almost no evidence, or evidence actively contradicts
Why this matters — Intellectual honesty is the core principle. A verdict of UNVERIFIABLE with 45% confidence is always more honest than a false REAL verdict at 99%.

Model Fallback & Availability Logic

Automatic cascade · 10 models · Zero downtime

If the primary model (Gemini 2.0 Flash) fails due to rate limits (HTTP 429) or unavailability (HTTP 404), the system automatically cascades through a priority-ranked list of 9 fallback models. Each model is tried in order until a valid response is received.
